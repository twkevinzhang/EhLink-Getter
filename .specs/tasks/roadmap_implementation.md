# EhLink-Getter V2 Transformation

**Status**: Planning
**Type**: Feature / Refactor
**Complexity**: XL
**Source**: .specs/analysis/roadmap.md, .specs/analysis/ui_blueprints.md

## Description

Transform the existing EhLink-Getter link crawler into a comprehensive desktop application for comic download and management ("EhLink-Getter V2").

## Objectives

1.  **Request Management**: Implement robust networking in Sidecar (Proxy pool, concurrency control, retries).
2.  **Task Management**: Create a multi-stage pipeline UI (Fetch -> Review -> Download) in Electron.
3.  **Library Management**: Build an E-Hentai themed local gallery viewer with advanced search.
4.  **UI/UX**: Adopt strict E-Hentai visual theme and maintain v0.0.1 layout metrics.

## Phases (per Roadmap)

1.  **Sidecar 2.0**: Backend networking and download capabilities.
2.  **Task Management**: Frontend task pipeline and persistence.
3.  **Library Management**: Metadata search and gallery grid UI.
4.  **Polish**: Integration and legacy cleanup.
